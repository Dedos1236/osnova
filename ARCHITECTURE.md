# Архитектурный отчет "НИТЬ" (Nit Messenger)

## 1. Подтверждение и Оценка (Главный Инженер)
Приветствую. Концепт мессенджера "НИТЬ" амбициозен, жесток к железу и требует экстремальной инженерной дисциплины. Архитектура 4-слойной гибридной сети с бесшовной миграцией поверх Multipath QUIC и аппаратным KVM-Mesh — это система класса *mission-critical*. Мы проектируем не просто мессенджер, а сурвайвл-сеть.

**Ключевые узкие места (Bottlenecks) в C++:**
1. **JNI / C-Interop Penalty на слое L2:** Получать IMU-кватернионы на частоте 200 Гц из Kotlin/Swift в C++ и синхронизировать их для миллисекундных BLE/Wi-Fi TX/RX окон крайне сложно. Оверхед вызова JNI может убить тайминги. Потребуется пакетирование данных (batching) и жесткая привязка к системному клоку (`CLOCK_MONOTONIC`).
2. **Congestion Control & Multipath QUIC State:** Классические алгоритмы контроля перегрузки (BBR, CUBIC) сломаются при мгновенном переключении с 4G (L1) на ультразвук (L3) с радикальным падением пропускной способности. Нам нужен кастомный state-machine для ngtcp2, мгновенно сбрасывающий окна перегрузки при падении слоя.
3. **Акустический Резерв (L3) DSP Pipeline:** Вычисления FFT/IFFT для FSK/PSK демодуляции в реальном времени съедят батарею. Потребуется максимальная утилизация ARM NEON инструкций на мобильных устройствах.

## 2. Структура Репозитория (Skeleton)
Единое C++23 ядро с жестким разделением ответственности.

```text
nit_messenger/
âCMakeLists.txt               # Корневой билд
âARCHITECTURE.md              # Документация системы
âinclude/                     # Публичные заголовки ядра (для мостов)
â   ânit/core/
âsrc/                         # Исходники единого C++ ядра
â   âcore/
â   â   ârouting/             # L3/L4 Балансировщик, алгоритмы
â   â   âmesh/                # DTN, P2P, Atomic Ledger (SQLite)
â   â   âcrypto/              # Noise Protocol, PQ-Crypto
â   â   âdsp/                 # Акустический модем (C++ NEON FFT)
â   â   âquic/                # ngtcp2 / Asio обертки
âmobile_bridges/              # Тонкие нативные обертки
â   âandroid/                 # Kotlin/JNI (Только Sensor Hub и BLE API)
â   âios/                     # Swift/C-interop (CoreBluetooth, CoreMotion)
âserver/                      # С++ Сервер (QUIC termination, роутинг, SQLite)
â   âsrc/
âui/                          # Графические интерфейсы
â   âdesktop/                 # Qt6/QML
â   ânative/                  # Мобильные UI (Jetpack Compose / SwiftUI) - опционально
âtests/                       # CTest/GTest
âscripts/                     # CI/CD
```

## 3. Roadmap (Декомпозиция на 6 Epics)

*   **Epic 1: Foundation & Transport (Ядро)**
    *   Настройка CMake, интеграция Asio и `msquic` (или `ngtcp2`). Реализация базового клиента и сервера C2S.
    *   *Артефакт:* Консольный echo-клиент/сервер через защищенный QUIC-туннель.
*   **Epic 2: The Core Ledger (Автономное Хранение)**
    *   Внедрение SQLite. Реализация транзакционного Atomic Ledger (fsync + rename) для DTN. Асинхронная очередь сообщений.
    *   *Артефакт:* Сервис сообщений, гарантированно выживающий при `kill -9` и сохраняющий стейт оффлайн-писем.
*   **Epic 3: Crypto-Layer & Blind Routing**
    *   Интеграция Noise Protocol. Разработка луковичной маршрутизации (многослойное шифрование пакетов) для транзитных узлов. Реализация серверного escrow для глобальных сессий (СОРМ).
    *   *Артефакт:* Обмен сообщениями, где промежуточные (mock) relay-узлы видят только крипто-мусор.
*   **Epic 4: AI-Balancer & L2 KVM-Mesh**
    *   Реализация `RoutingBrain`. Сборка JNI/C-interop для `Sensor Hub` (IMU 200Hz). Интеграция BLE Coded PHY. Логика кинетической векторной модуляции.
    *   *Артефакт:* Два смартфона успешно обмениваются ключами и сообщениями по BLE без LTE/Wi-Fi интерфейсов во время вращения устройства.
*   **Epic 5: Autonomous Zero-Net & DTN (L4)**
    *   Активация Broadcast (Swarm Mode). Шлюзование. Если один узел поймал 4G, он становится Gateway для всей offline-сети, скидывая накопленный Ledger.
    *   *Артефакт:* Сценарий блэкаута: телефон A в лесу передает пакет телефону B (на краю леса), который выходит в 4G и доставляет письмо серверу.
*   **Epic 6: L3 Acoustic Reserve & Qt6 UI Integration**
    *   Написание DSP (NEON) акустического модема. Сборка десктопного UI на Qt6/QML и подключение ядра к тонким мобильным UI.
    *   *Артефакт:* Production-ready билды (APK, IPA, EXE/AppImage). Успешная передача ключа по ультразвуку.

В корне файловой системы созданы `CMakeLists.txt` и базовый заголовочный файл `routing_brain.h` ядра.
